How it works

LLM reasons. JavaScript acts.

The model never touches the DOM or Chrome APIs. It writes JavaScript; a sandboxed runtime turns that into typed page.* commands against the active tab.

01

You describe the task

“Find the cheapest direct flight from SFO to JFK next Friday and fill the search form.”

02

The agent reasons + generates JS

It snapshots the page, plans, and emits JavaScript via its run_js tool — no direct browser access.

03

The runtime executes page.* commands

@pi-oxide/extension-js runs the JS and dispatches typed BrowserCommands to the active tab.

04

It observes + iterates

The result feeds back to the model. It adjusts, retries on error, and stops when the task is done.

What it does

Built for transparent browser automation.

Plain-English tasks

Type what you want. The agent reasons with Claude, plans steps, and recovers from errors on its own.

Sees the page

An accessibility-tree snapshot gives the model structured elements — refs, roles, labels — not pixels.

Acts through run_js

The LLM's only tool is run_js. It writes JS that calls page.* APIs; it never touches the DOM or Chrome directly.

Bring your own key

No inference markup. Point at Anthropic, DeepSeek, GLM, or any Anthropic-compatible endpoint.

Skill-aware

Activate reusable skills with /skill: and attach session files with @[file:…] — composed into the prompt.

Inspectable by design

Every step is traced: the generated JS, the result, the error. Nothing happens in a black box.

Quick start

Run it from source in a minute.

git clone https://github.com/Irvingouj/Browsergent.git
cd Browsergent
npm install
npm run build

Then open chrome://extensions, enable Developer mode, and Load unpacked the dist/ directory. Or grab the prebuilt release on the download page.
